# Pundo

## 植物空間點位標記工具 Plant Spatial Points Tool

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.xxxxxxx.svg)](https://doi.org/10.5281/zenodo.xxxxxxx)

[**Pundo**](https://jjakon11.github.io/Pundo/) 是一款專為森林生態學、森林動態樣區調查與植物空間資料建檔所設計的輕量級網頁工具。研究人員可以在樣區底圖上快速標記樹木或木本植物的空間位置，並記錄物種、胸高直徑（DBH）等重要調查資訊。

Pundo 的核心概念是以森林樣區的西南角，或圖面左下角，作為相對座標原點 **(0, 0)**，再依照每棵植物在樣區中的相對位置與距離進行座標標定。

若使用者已知樣區西南角的大地座標，後續即可將每棵植物的相對座標進一步轉換為大地座標。此方法可減少逐棵植物使用 GPS 定位時可能產生的誤差，特別適合森林樣區、固定樣區與長期生態監測樣區使用。

---

## 主要功能

### 1. 純前端運行

Pundo 不需要資料庫、伺服器或額外安裝軟體。使用者只需透過支援 HTML5 的瀏覽器開啟工具，即可進行資料匯入、點位標記與資料輸出。

所有資料皆保留在使用者自己的電腦中，不會上傳至遠端伺服器，因此可提高調查資料的安全性與隱私性。

### 2. 支援底圖空間校正

使用者可以匯入 JPG、PNG、PDF 格式的樣區掃描圖、手繪圖或電子化底圖。

工具支援底圖邊界黑框辨識，並提供使用者微調邊界與旋轉角度設定功能。使用者可將圖紙上的樣區外框精準對齊數位畫布與座標網格，以提升後續點位標記的準確度。

### 3. 互動式點位標記

使用者可直接在畫布上以滑鼠點擊方式標記植物位置。系統會記錄該植物的空間座標，並可依照 DBH 數值自動調整點位圖示大小，使植株空間分布與大小結構更直觀呈現。

### 4. 一鍵匯出 CSV

完成點位標記後，使用者可直接將資料下載為標準 CSV 格式。匯出的資料可用於後續統計分析、空間分析、GIS 製圖或森林動態樣區資料管理。

### 5. 批次匯出調查用 PDF

完成點位標記後，使用者可以將圖面批次匯出為 PDF 檔案，作為後續野外調查、資料核對或紙本紀錄使用。

## 使用流程

### 1. 資料上傳

若使用者已經有植物調查資料，可透過「資料上傳」功能匯入 CSV 檔案。系統支援 **UTF-8** 與 **Big5** 編碼格式。

以森林動態樣區資料為例，匯入資料可包含下列欄位：

| 欄位名稱   | 說明                 |
| ------ | ------------------ |
| `Tag`  | 植株編號，**必要欄位**      |
| `plot` | 樣區名稱               |
| `x1`   | 10 × 10 m 樣方的 X 座標 |
| `y1`   | 10 × 10 m 樣方的 Y 座標 |
| `x2`   | 5 × 5 m 子樣方的 X 座標  |
| `y2`   | 5 × 5 m 子樣方的 Y 座標  |
| `Sp`   | 物種名稱               |
| `DBH`  | 胸高直徑，單位依原始資料設定     |

其中，**`Tag` 為必要欄位**。若匯入資料中沒有 `Tag` 欄位，系統將無法進行後續點位標記與資料更新。

其他欄位則可依資料狀況彈性使用。例如，若使用者處理的是零散樣區或非森林動態樣區資料，即使資料中沒有 `x1`、`y1`、`x2`、`y2` 等欄位，仍可進入系統並進行點位標記。

匯入資料請參考[示範資料](https://github.com/jjakon11/Pundo/tree/main/Demo_file)

---

### 2. 畫布設定與底圖校正

進入系統後，使用者需先設定樣區邊界大小。樣區形狀不一定要是正方形，邊界長度單位為 **cm**，目前預設值為 **500 cm**。此設定會影響後續點位在畫布上的比例與座標計算結果。

使用者也可以上傳樣區手繪圖、掃描圖或電子化底圖。系統會辨識底圖中的黑框邊界。匯入的底圖格式可參考 [示範地圖](https://github.com/jjakon11/Pundo/tree/main/Demo_map)。

上傳底圖後，使用者可透過下列功能進行校正：

* 微調底圖邊界
* 調整底圖旋轉角度
* 對齊畫布上的數位網格
* 對齊樣區外圍黑框或實際調查邊界

完成校正後，底圖即可作為後續點位標記的參考。

---

### 3. 植株點位標記

確認目標植株資料無誤後，點擊「準備點擊圖面」按鈕，接著在畫布上的對應位置點擊滑鼠，即可完成該植株的空間座標標記。

若該 `Tag` 已存在，系統會更新該筆資料的 `x3` 與 `y3` 座標。

若該 `Tag` 不存在，系統會新增一筆資料，並記錄使用者輸入的 `plot`、`Sp`、`DBH` 與座標資訊。

系統會將點擊位置轉換為對應的空間座標，並將精細座標記錄於 `x3`、`y3` 兩個欄位中。這兩個欄位表示植株相對於樣方左下角或西南角的距離，單位為 **cm**。

若使用者沒有既有調查資料，例如正在建立新的樣區資料，也可以省略資料匯入步驟，直接進行畫布設定、底圖校正與植株點位標記。

---

### 4. 資料匯出

完成所有植物點位標記與資料確認後，使用者可點擊「下載完成的 CSV 檔案」按鈕。

匯出的 CSV 檔案將包含原始調查資料，以及新增或更新後的完整空間座標欄位，例如：

| 欄位名稱 | 說明             |
| ---- | -------------- |
| `x3` | 植株在樣區中的精細 X 座標 |
| `y3` | 植株在樣區中的精細 Y 座標 |

匯出後的資料可直接用於 R、Python、QGIS、ArcGIS 或其他統計與空間分析軟體。

---

### 5. PDF 檔案匯出

完成所有植物位置標記後，使用者可以批次輸出所有樣區的野外調查用 PDF 檔案，也可以自行選擇特定樣區進行輸出。

輸出的 PDF 可用於後續野外調查、資料核對、點位修正或紙本紀錄。

---

## 系統需求

Pundo 可於任何支援 HTML5 的現代網頁瀏覽器中執行，例如：

* Google Chrome
* Microsoft Edge
* Safari
* Firefox

使用本機版 HTML 檔案時，不需額外安裝軟體，也不需網路連線即可使用。

## 示範檔案與簡易流程

Pundo 軟體：https://jjakon11.github.io/Pundo/

示範資料：https://github.com/jjakon11/Pundo/tree/main/Demo_file

示範地圖：https://github.com/jjakon11/Pundo/tree/main/Demo_map

簡易示範流程如下：

1. 下載 [示範資料](https://github.com/jjakon11/Pundo/tree/main/Demo_file)。

2. 下載 [示範地圖](https://github.com/jjakon11/Pundo/tree/main/Demo_map)。

3. 開啟 [Pundo 軟體](https://jjakon11.github.io/Pundo/)。

4. 在 **1. 資料上傳** 中上傳示範資料。

5. 在 **2. 點位植株** 中選擇不同的 `Plot`，或在 **4. 資料呈現** 中檢視整份資料。

6. 選擇想要操作的 `Plot`，例如 `Demo_plot1`，再依序選擇：

   * `x1 = 10`
   * `y1 = 3`
   * `x2 = 1`
   * `y2 = 1`

7. 在 **3. 畫布與底圖校正** 中上傳底圖，並選擇 `DemoPlot1_(10,3)(1,1).pdf` 檔案。

8. 此樣方中有三個植株尚未具有座標：

   * `Y25333`
   * `Y25334`
   * `Y25335`

9. 在 **2. 點位植株** 的 `Tag` 欄位中輸入目標植株編號。

10. 按下「準備點擊圖面」後，在畫布內對應位置點擊滑鼠。

11. 完成植株點位標記。

## 適用情境

Pundo 適合用於下列工作：

* 森林樣區手繪圖數位化
* 木本植物空間點位標記
* 森林動態樣區每木資料建檔
* 樣區座標資料整理
* 相對座標轉換為大地座標前的資料準備
* 長期生態監測樣區資料管理
* 野外調查資料後處理

本工具後續亦透過 AI 協作工具共同開發與改良，包括：
